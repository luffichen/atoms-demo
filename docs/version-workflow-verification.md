# 版本工作流专项验收

验收日期：2026-07-20

## 覆盖范围

- 需求 91–96：放弃快照、版本历史、版本详情、历史代码、工作记录、预览联动。
- 需求 101：固定视口、触控、深链接、性能和发布中断恢复。

## 自动化回归

- `server/app.test.ts` 验证结构化工作放弃后保留原因、归档时间、工作分支、快照文件树和相对基线 Diff，并允许创建下一工作项。
- `src/components/VersionViewer.test.tsx` 验证当前工作默认入口、六个页签、锁定状态、版本化文档只读查看、正式版本元数据、文件路径深链接及历史工作对话。
- 发布执行状态由服务端持久化为 `running`，项目刷新从权威快照恢复；现有“发布事务进行中可从服务端快照看到进度并在完成后收敛”回归测试覆盖刷新恢复。

## 固定视口验收

使用本地真实浏览器打开活动结构化需求的“版本 → 当前工作”，检查 390、768、1024 和 1440 像素宽度。

| 宽度 | 结果 |
| --- | --- |
| 390 | 单列；页面 `scrollWidth = 390`；版本次级入口和六个页签最小高度 44px。 |
| 768 | 单列；页面 `scrollWidth = 768`；版本面板宽度 768px；交互项最小高度 44px。 |
| 1024 | 紧凑双栏；页面 `scrollWidth = 1024`，无页面级横向溢出。 |
| 1440 | 紧凑双栏；页面 `scrollWidth = 1440`，无页面级横向溢出。 |

键盘焦点沿用原生按钮、页签和只读文件区域；版本状态、加载状态和快照横幅具有语义标签。

## 性能门槛

在 Fastify 注入环境对完整路由、SQLite 查询和 Git tree/blob/diff 读取各采样 20 次，以第 19 个有序样本作为 p95。测试会在 CI 中持续断言相同门槛。

| 场景 | p95 | 门槛 |
| --- | ---: | ---: |
| 当前工作首屏 | 0.5ms | < 2,000ms |
| 版本历史首屏 | 0.1ms | < 2,000ms |
| 工作记录首屏 | 0.1ms | < 2,000ms |
| 历史文件树 | 9.8ms | < 2,000ms |
| 单文件 Diff | 9.9ms | < 3,000ms |

说明：这是可重复的本机服务端性能门槛，不包含公网传输时间；页面同时保留加载、空、错误和禁用状态。

## 结论

需求 91–96、101 的功能与专项验收通过。
