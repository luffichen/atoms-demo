# 实时事件延迟专项验收

验收日期：2026-07-20（Asia/Shanghai）

## 结论

需求 63、68 通过。正常本地网络的可控采样中，消息状态、用户可见文字分片、文件写入、终端输出和 Todo 更新的 WebSocket 交付及页面可见 p95 均小于 1 秒；所有样本均在 1 秒内完成。

## 采样边界

- 起点是事件交给 `RealtimeHub.publish`，或浏览器 WebSocket 收到同一事件；不包含模型思考、命令执行和文件操作自身耗时。
- WebSocket 层使用真实 Fastify WebSocket 路由、序列化和客户端接收，各类 30 个样本。
- 页面层使用项目页真实事件处理与 React DOM 提交，各类 20 个样本；断言目标内容已经出现在用户可见 DOM。
- 流式文字连续向同一 assistant item 追加；重连完整性另由项目 WebSocket 权威 `sync` 快照回归测试覆盖。
- 除原需求规定的消息状态、文字、文件和终端外，额外采样 Todo 更新。
