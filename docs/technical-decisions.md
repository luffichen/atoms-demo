# 技术方向修订记录

本文件只记录访谈中明确提出的技术方向变化，不属于任务需求文档。

## TD-001 使用项目 ID 隔离项目目录

- 状态：已确认
- 原始方向：游客项目目录使用项目名称作为路径的一部分。
- 修订方向：每个项目拥有唯一项目 ID，项目文件目录使用项目 ID 作为路径的一部分。
- 原因：项目显示名称允许重复，也可能包含不适合作为路径的字符；显示名称不应承担资源身份和路由职责。
- 影响：飞书原文保留为历史归档，后续实现应以本决策为准。

## TD-002 使用游客 ID 隔离游客目录

- 状态：已确认
- 原始方向：游客根目录直接使用游客名称。
- 修订方向：每个游客拥有唯一且稳定的游客 ID；游客根目录使用游客 ID，项目目录再使用项目 ID。
- 原因：游客名称只承担展示作用，不应成为文件身份或数据隔离边界。
- 影响：项目目录的身份路径由游客 ID 与项目 ID 共同确定；游客名称大小写规则不影响已有资源。

## TD-003 限制智能体的公网访问边界

- 状态：已确认
- 方向：智能体可访问公网，用于安装公开依赖和读取公开资源。
- 禁止范围：宿主机内部服务、其他游客目录、云平台元数据地址及任何预置密钥。
- 可见性：网络相关命令与结果作为终端事件向用户展示。

## TD-004 将智能体限制在当前项目环境

- 状态：已确认
- 方向：智能体只能读写当前项目目录，并运行当前项目所需的普通进程。
- 禁止范围：其他项目、其他游客、宿主机文件和特权进程。
- 失败行为：越界操作必须被拒绝，并以失败事件向用户展示。

## TD-005 隔离生成应用预览

- 状态：已确认
- 方向：生成网页在独立安全边界中预览。
- 禁止范围：读取或修改 Atoms Demo 主应用的状态、游客选择、页面内容和浏览器存储；跳转或控制主页面。
- 故障边界：预览崩溃不得影响对话、任务队列或其他工作区功能。

## TD-006 限制生成应用的公网访问

- 状态：已确认
- 方向：生成应用可请求公开 HTTPS API 和资源。
- 禁止范围：本机、内网、云平台元数据地址和 Atoms Demo 主应用凭据。
- 故障边界：生成应用的网络失败只影响其自身预览。

## TD-007 首版不管理项目秘密

- 状态：已确认
- 方向：首版不提供 API Key、Token 或项目环境变量的输入与管理。
- 安全要求：智能体不得要求用户在对话中粘贴秘密。
- 产品边界：依赖私有凭据才能运行的应用不属于首版可成功预览范围。

## TD-008 允许隔离范围内自动执行

- 状态：已确认
- 方向：智能体在当前项目允许范围内执行普通文件操作和终端命令时，无需用户逐次批准。
- 可见性：所有操作继续实时展示为文件或终端事件。
- 禁止行为：越界或明确禁止的操作直接拒绝并显示失败，不弹出审批流程。

## TD-009 限制项目磁盘空间

- 状态：已由 TD-026 取代
- 上限：每个项目最多使用 1 GB 磁盘空间。
- 超限行为：停止当前写入并将任务标记失败，提示“项目空间已满”。
- 保留能力：已有文件继续可查看，用户仍可通过后续对话要求智能体清理文件。

## TD-010 不设置每项目计算资源配额

- 状态：已确认
- 方向：首版不为单个项目单独限制 CPU 核心数或内存用量。
- 仍然生效：单条消息 30 分钟执行时限及项目隔离边界；项目 1 GB 磁盘限制已由 TD-026 取消。

## TD-011 不限制实例级任务并发

- 状态：已确认
- 方向：首版不限制整个演示实例同时执行的项目任务数量。
- 仍然生效：同一项目任意时刻只执行一条消息，项目内其他消息串行排队。

## TD-012 使用项目 ID 关联唯一会话

- 状态：已由 TD-023 取代
- 方向：一个项目只对应一条持续对话，不额外生成独立 session ID。
- 目录关系：项目文件与会话分开存放，但二者均使用同一个项目 ID 作为目录标识。
- 结果：无需维护项目 ID 与 session ID 的一对一映射。

## TD-013 部署到单台 Google Compute Engine

- 状态：已确认
- 原始表述：“google 的 ecs 实例”。
- 修订方向：首版部署到单台 Google Compute Engine（GCE）虚拟机。
- 说明：原文中的 ECS 视为 GCE 笔误，不采用 AWS ECS。

## TD-014 使用持久化工作磁盘

- 状态：已确认
- 方向：`/workspace` 位于持久化磁盘。
- 保证：GCE 虚拟机重启或应用重新部署不得清空游客、项目、排队消息、会话历史和项目文件。
- 中断边界：重启可以按已确认规则中断当前执行消息，但不能丢失其前置数据。

## TD-015 首版不提供自动备份

- 状态：已确认
- 方向：不实现自动备份或灾难恢复。
- 保证范围：仅保证持久化磁盘在普通虚拟机重启和应用部署后保留数据。
- 非目标：磁盘损坏或管理员误删后的自动恢复。

## TD-016 公网开放无认证演示实例

- 状态：已确认
- 方向：Atoms Demo 直接对公网开放，不限制为内网、VPN 或 IP 白名单。
- 公开边界：所有游客均可被任何访问者选择，项目和对话不得视为私密数据。
- 风险提示：不得在实例中输入敏感内容或真实凭据。

## TD-017 首版不做请求限流

- 状态：已确认
- 方向：首版不按 IP 或游客限制项目创建和消息发送频率。
- 已接受风险：公网实例可能遭受资源滥用；首版不提供产品内缓解措施。

## TD-018 公网访问强制 HTTPS

- 状态：已确认
- 方向：所有公网访问使用 HTTPS，HTTP 请求自动跳转到 HTTPS。
- 适用数据：图片、对话、文件内容、终端输出和应用预览流量。

## TD-019 自动停止空闲预览

- 状态：已确认
- 方向：网页项目最后一个查看者离开 10 分钟后，自动停止预览进程。
- 恢复：再次打开“应用”时自动启动，并显示“正在启动预览”。
- 隔离：停止预览不影响项目文件、对话、智能体任务或消息队列。

## TD-020 使用服务端统一默认模型

- 状态：已确认
- 方向：所有游客使用服务端统一配置的一个默认模型。
- 产品边界：界面不提供模型查看、选择或切换，也不保存项目级模型偏好。
- 配置边界：具体模型由部署配置指定。

## TD-021 首版不接入产品分析

- 状态：已确认
- 方向：不实现用户行为埋点、使用分析或第三方错误上报。
- 保留范围：仅保留服务正常运行和排障所需的普通日志。

## TD-022 使用平台托管的项目级 Git 仓库

- 状态：已确认
- 方向：每个项目创建独立 Git 仓库，Git 元数据位于智能体不可写的管理目录。
- 权限：commit、分支、merge 和 tag 只由平台执行；智能体只能通过受控能力读取状态与 Diff。
- 正式版本：项目内使用 `V1、V2…`，tag 使用 `code/v1、code/v2…`。
- 产品边界：首版不连接远程 Git，不提供版本恢复、删除、重命名、下载或任意比较。

## TD-023 使用工作项隔离对话与 session

- 状态：已确认
- 方向：项目下每项结构化需求或直接编码工作均创建独立工作项。
- 会话：每个工作项拥有独立对话、终端记录和 `sessions/{projectId}/{workItemId}` JSONL 目录。
- 连续性：同一工作项内跨阶段或多轮消息继续同一 session；不同工作项绝不自动续接。
- 并发：首版每个项目同一时刻最多一个活动工作项。
- 修订：本决策替代 TD-012 中“一个项目只对应一条持续对话”的方向。

## TD-024 结构化需求研发为可选模式

- 状态：已确认
- 方向：项目工作分为“结构化需求”和“直接编码”。
- 结构化需求：执行逐题访谈、需求文档、技术方案、开发、测试、候选预览和确认上线。
- 直接编码：收到消息后立即允许修改代码，多轮改动在用户确认发布时形成一个正式版本。
- 默认：新项目默认直接编码，用户可以显式选择按需求研发。

## TD-025 正式版本只在用户确认发布后产生

- 状态：已确认
- 方向：内部 commit 只作为阶段或编码检查点，不显示为正式版本。
- 结构化来源：测试和候选预览通过后，用户二次确认上线。
- 直接编码来源：轻量验证通过后，用户二次确认发布。
- 历史：正式版本及其文档、tag 和发布记录不可修改或删除。

## TD-026 取消项目空间总限制

- 状态：已确认
- 原始方向：TD-009 规定每项目最多使用 1 GB。
- 修订方向：移除每项目 1 GB 配额、项目容量预警和实例级磁盘低水位保护。
- 保留限制：单张上传图片和单个进入 Git 的二进制文件仍不得超过 10 MB。
- 已接受风险：公网演示实例可能因项目、版本或附件持续增长而耗尽存储。
- 影响：TD-009 不再适用于新版本实现。

## TD-027 图片使用逐文件独立上传

- 状态：已确认
- 方向：取消每条消息 5 张图片和消息图片总大小限制；每张图片独立上传，消息只提交附件 ID。
- 单文件限制：只支持 PNG/JPEG，单张不超过 10 MB。
- 生命周期：未发送临时附件保留 24 小时；正式对话附件长期保留。
- 文档资产：被需求或测试文档引用的图片复制到仓库文档资产目录。

## TD-028 新版本不兼容历史数据

- 状态：已确认
- 方向：功能上线前删除现有项目、对话、session、附件和工作区数据。
- 结果：不实现旧会话到工作项、旧项目到 Git、旧代码到迁移 `V1` 的兼容逻辑。
- 风险：这是破坏性切换，应用代码回滚不能恢复已删除历史数据。

## TD-029 使用 PrismJS 实现共享文件语法高亮

- 状态：已确认
- 方向：当前项目、实时写入和历史版本文件使用同一 React 只读查看器；PrismJS
  仅加载明确支持的语言模块，并把 token 转为 React 节点。
- 安全：不使用 `dangerouslySetInnerHTML`；未知语言和解析异常均安全降级为纯文本。
- 性能：超过 200 KB 或 5,000 行关闭高亮；实时写入约 100 ms 防抖刷新。
- 视觉：语法 token 参考 GitHub Light，查看器结构与交互继续使用项目 Atoms tokens。
- 产品边界：不引入完整代码编辑器、自动语言猜测、主题切换或 Diff 高亮。
