# 适配项目预览能力

## 目标

工作区和项目卡片根据项目是否具备浏览器预览能力展示正确入口，不把运行失败误判为非网页项目。

## 需求

1. 项目是否为网页项目，按其是否声明并具备浏览器启动方式判断。
2. 具备启动方式但预览失败时，仍视为网页项目并保留“应用”模式。
3. 没有浏览器启动方式时，只显示“文件”和“终端”。
4. 非网页项目卡片显示统一代码项目占位图。
5. 非网页项目后续具备预览能力时，显示“应用”模式。
6. 网页项目后续变为非网页项目时，隐藏“应用”并停止展示旧预览。
7. 项目再次成为网页项目时，必须产生新的成功预览后才展示，不恢复过时预览。

## 验收标准

- 网页预览启动失败时仍可进入“应用”看到失败提示。
- 非网页项目看不到“应用”，卡片显示代码项目占位图。
- 项目在两类之间变化时，入口和卡片同步正确变化。
- 从非网页重新变为网页后，旧预览不会提前出现。

## 不在本任务范围内

- 刷新或导航应用预览。
- 生成项目代码。
- 项目卡片其他信息。

## 独立性与原子性检查

- **单一职责**：只让界面跟随项目预览能力变化。
- **可独立验证**：使用网页、预览失败、非网页和类型转换四类场景即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
