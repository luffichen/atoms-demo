# 自动生成项目名称

## 目标

新项目创建时，系统根据用户在首页提交的第一条需求生成一个简短、易识别的显示名称，方便用户在项目列表中找到它。

## 前置条件

- 用户已经提交一条可用于创建项目的有效需求。

## 需求

1. 系统根据第一条需求的核心主题自动生成项目名称。
2. 项目名称最多包含 20 个可见字符。
3. 项目名称不得直接复制整段需求。
4. 项目名称统一使用中文。
5. 不要求同一游客的项目名称互不重复。
6. 如果名称生成失败，项目仍应正常创建，并使用“新项目”作为名称。
7. 项目创建后，名称保持不变；后续对话不会自动改变名称。
8. 首版不提供项目重命名入口。

## 验收标准

- 给定一条简短需求，创建出的项目具有能够概括需求主题的名称。
- 给定一条超过 20 个可见字符的需求，生成的名称不超过 20 个可见字符，且不是整段需求的直接复制。
- 中文、英文和混合语言需求均产生中文项目名。
- 连续创建主题相同的项目时，即使名称相同，也都能成功创建。
- 名称生成失败时，项目能够成功创建，名称显示为“新项目”。
- 在项目中继续提交修改需求后，原项目名称保持不变。
- 用户在首版界面中找不到重命名项目的入口。

## 不在本任务范围内

- 创建项目。
- 校验首页输入。
- 启动或执行生成任务。
- 展示或管理项目列表。
- 删除、归档或重命名项目。
- 定义项目的存储位置。

## 独立性与原子性检查

- **单一职责**：本任务只决定新项目显示名称的产生和兜底行为。
- **可独立验证**：可以输入不同的首条需求并观察名称结果，无需验证智能体生成应用的过程。
- **边界明确**：项目创建、项目列表和生成执行均由其他任务负责。
- **结论**：任务足够独立且原子，无需继续拆分。
