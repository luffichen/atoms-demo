# 展示项目对话内容

## 目标

用户在左侧对话区能够理解自己提出了什么、智能体如何回应，以及每轮任务处于什么结果。

## 需求

1. 时间线展示用户消息。
2. 时间线展示智能体面向用户的文字回复。
3. 时间线展示每条用户消息的任务状态。
4. 用户消息中的图片以可识别附件形式展示。
5. 不展示智能体内部推理、隐藏提示、系统上下文或其他仅供内部使用的内容。
6. 不把原始内部事件直接作为普通文本倾倒到对话中。

## 验收标准

- 重新打开项目后可以按顺序看到用户消息和面向用户的回复。
- 含图片的消息能够识别其图片附件。
- 排队、执行、完成、失败和取消状态显示在对应消息附近。
- 内部推理、隐藏提示和系统上下文均不会出现在页面中。

## 不在本任务范围内

- 文件和终端事件卡片。
- 历史分批加载。
- 消息排队和状态推进。
- 图片放大查看。

## 独立性与原子性检查

- **单一职责**：只定义普通对话内容的可见范围。
- **可独立验证**：使用包含不同角色、图片和隐藏内容的历史即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
