# 取消排队消息

## 目标

用户可以撤销尚未开始的修改要求，避免智能体执行已经不需要的任务。

## 需求

1. 每条“排队中”消息提供“取消”操作。
2. 取消操作无需二次确认。
3. 取消后，该消息状态变为“已取消”且永不执行。
4. 其他排队消息保持原有相对顺序。
5. 取消立即释放一个队列名额。
6. 队列原本已满时，取消后恢复发送能力。
7. 首版不支持编辑已排队消息。

## 验收标准

- 取消任意一条排队消息后，该消息不会进入执行状态。
- 取消队列中间的消息后，前后消息仍按原相对顺序执行。
- 满队列取消一条后，用户可以立即发送一条新消息。
- 执行中、已完成、失败或已取消的消息不显示排队取消操作。
- 页面中没有编辑排队消息的入口。

## 不在本任务范围内

- 停止正在执行的消息。
- 删除消息历史。
- 修改排队顺序。

## 独立性与原子性检查

- **单一职责**：只撤销一条尚未执行的消息。
- **可独立验证**：取消队首、队中和满队列中的消息即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
