# 分批加载对话历史

## 目标

长期项目重新打开时，可以快速看到最近活动，并按需阅读更早内容。

## 需求

1. 初次打开项目先加载最近 50 条时间线内容。
2. 初次加载成功后定位到时间线底部。
3. 还有更早内容时，顶部显示“加载更早内容”。
4. 每次操作再加载 50 条更早内容。
5. 追加更早内容后，保持用户当前阅读位置不跳动。
6. 没有更早内容后，不再显示加载入口。
7. 加载失败时保留当前已显示内容，在顶部显示失败提示和“重试”。
8. 加载失败不影响正在执行的任务或新活动实时更新。

## 验收标准

- 超过 50 条的项目首次只展示最新 50 条并位于底部。
- 每次加载更早内容追加最多 50 条，顺序正确且无重复。
- 追加后用户原本看到的内容仍停留在相同位置。
- 加载失败时已有内容不丢失，实时新活动仍然出现。
- 重试成功后从正确位置继续加载。

## 不在本任务范围内

- 对话内容样式。
- 对话底部自动跟随。
- 搜索历史。

## 独立性与原子性检查

- **单一职责**：只负责读取并向上追加历史时间线内容。
- **可独立验证**：使用超过 100 条内容并制造一次失败即可完整覆盖。
- **结论**：任务足够独立且原子，无需继续拆分。
