# 限制项目存储空间

> 状态：已由需求 99 和 TD-026 取消。新版本不实现每项目 1 GB 配额、容量预警或实例级低水位保护；本文仅保留为历史需求记录。

## 目标

单个项目不能无限占用演示实例空间；达到上限后项目仍保持可查看和可清理状态。

## 需求

1. 每个项目最多使用 1 GB 存储空间。
2. 达到上限时停止导致超限的写入。
3. 当前消息标记为“失败”，并提示“项目空间已满”。
4. 超限前已经产生的文件继续保留且可查看。
5. 用户仍可发送消息，要求智能体删除不需要的文件。
6. 清理出空间后，后续任务可以继续正常写入。

## 验收标准

- 项目未到上限时能够正常写入。
- 尝试越过 1 GB 时当前任务失败且原因明确。
- 已有文件未丢失，文件和终端查看仍可使用。
- 清理文件后可以再次执行写入任务。

## 不在本任务范围内

- 限制项目数量。
- 自动删除文件。
- 限制 CPU 或内存。

## 独立性与原子性检查

- **单一职责**：只限制一个项目的磁盘占用。
- **可独立验证**：在上限前、越界和清理后三个阶段即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
