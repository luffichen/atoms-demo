# 停止正在执行的消息

## 目标

用户可以终止不再需要或耗时过长的当前任务，并让项目队列继续运行。

## 需求

1. 项目存在“执行中”消息且输入框没有非空文字时，输入框右下角的发送按钮切换为“停止”；执行中消息卡片内不展示停止操作。
2. 点击“停止”后先显示确认框，说明文件变化不会回滚且后续队列继续执行。
3. 用户取消确认时，当前任务保持运行。
4. 用户确认并停止成功后，消息状态变为“已取消”。
5. 停止前已经产生的项目文件变化继续保留。
6. 系统不自动回滚任何文件变化。
7. 停止完成后，自动开始下一条排队消息。
8. 非执行中状态下输入框不显示“停止”操作。
9. 执行中输入框出现非空文字时，“停止”切换回“发送”，新消息按排队协议提交。
10. 队列已满不影响空输入框状态下停止当前任务。

## 验收标准

- 停止执行中消息后，智能体不再继续该任务，状态显示“已取消”。
- 取消确认时任务不受影响。
- 停止前已经写入的文件仍可查看。
- 项目中没有发生自动回滚。
- 有后续排队消息时，下一条自动开始；没有时队列进入空闲。
- 重复点击停止不会造成多个状态变化或启动多条后续消息。
- 执行中空输入框只在输入框主操作位显示停止；消息卡片内没有停止入口。
- 输入文字后主操作切换为发送，清空文字后恢复停止。

## 不在本任务范围内

- 取消排队消息。
- 恢复被停止的任务。
- 手动回滚文件。

## 独立性与原子性检查

- **单一职责**：只终止当前一条执行中消息并推进队列。
- **可独立验证**：在任务产生文件变化前后分别停止即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
