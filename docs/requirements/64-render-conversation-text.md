# 渲染对话文本

## 目标

智能体回复具有清晰格式，同时用户原始需求不因格式解析而改变含义。

## 需求

1. 智能体回复支持段落、标题、列表、引用、链接、行内代码和代码块等常用 Markdown。
2. 智能体回复中的 HTML 或脚本不得执行。
3. 智能体回复中的外部链接在新标签页打开。
4. 用户消息按纯文本显示并保留换行。
5. 用户消息中的 Markdown 或 HTML 不解析。
6. 时间线不显示日期分隔线。

## 验收标准

- 智能体常用 Markdown 正确显示。
- HTML 和脚本仅作为安全内容处理，不执行。
- 外部链接打开新标签页。
- 用户输入的符号、标签和换行按原意显示。
- 跨日期消息之间没有额外日期分隔条。

## 不在本任务范围内

- 流式传输智能体文字。
- 文件语法高亮。
- 图片附件预览。

## 独立性与原子性检查

- **单一职责**：只决定两类对话文本的呈现规则。
- **可独立验证**：使用 Markdown、HTML 和纯文本样例即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
