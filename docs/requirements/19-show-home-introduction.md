# 展示首页引导内容

## 目标

用户进入首页后，能立即理解产品用途，并确认当前正在使用哪个游客。

## 需求

1. 首页主区域显示“输入想法，生成应用。开始吧，{游客名}。”。
2. `{游客名}` 替换为首页当前游客的名称。
3. 首页切换游客后，主标题立即显示新游客名称。
4. 需求输入框提示文案为“描述你想创建的应用”。
5. 首版不展示轮播标语或运营通知。
6. 首版不展示语音输入或智能体生成的建议按钮。

## 验收标准

- 使用 `default` 进入首页时，标题包含 `default`。
- 切换到其他游客后，标题中的名称同步变化。
- 输入框未输入内容时显示约定提示文案。
- 页面中没有轮播标语或运营通知区域。

## 不在本任务范围内

- 游客切换操作本身。
- 输入校验、附件或提交需求。
- 首页侧边栏导航。

## 独立性与原子性检查

- **单一职责**：只展示首页的用途说明和当前游客信息。
- **可独立验证**：使用两个游客切换首页展示数据即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
