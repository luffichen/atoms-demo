# 跟随对话最新活动

## 目标

用户关注执行进度时自动看到最新活动；阅读历史时不会被实时更新强制拉回底部。

## 需求

1. 用户位于对话底部时，时间线自动跟随最新消息和事件。
2. 用户主动向上滚动后，停止自动滚动。
3. 暂停跟随时显示“有新活动”。
4. 点击“有新活动”回到底部并恢复自动跟随。
5. 新活动到达时不得改变用户正在阅读的历史位置。

## 验收标准

- 停留底部时连续事件始终可见。
- 向上滚动后，新事件不会改变当前阅读位置。
- 暂停期间出现“有新活动”。
- 点击提示后回到底部，后续事件继续自动跟随。

## 不在本任务范围内

- 文件和终端查看器滚动。
- 历史分批加载。
- 消息或事件内容。

## 独立性与原子性检查

- **单一职责**：只控制对话区是否跟随底部。
- **可独立验证**：在底部和历史位置分别制造新事件即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
