# 标记本轮变化文件

## 目标

用户可以从文件树快速识别当前或最近一轮任务新建和修改了哪些文件。

## 需求

1. 标记范围相对于最近一条已经开始执行的用户消息。
2. 本轮首次产生的文件标记“新建”。
3. 本轮开始前已存在、随后发生变化的文件标记“更新”。
4. 下一条消息开始执行时重置上一轮标记。
5. 重新打开项目时展示当前执行轮或最近完成一轮的标记。
6. 标记不能只依赖颜色区分。
7. 已有文件重命名或移动后，在新路径标记“更新”，旧路径消失。
8. 本轮新建文件随后重命名或移动时，仍标记“新建”。

## 验收标准

- 同一轮新文件和已有修改文件获得正确、可辨认的标记。
- 下一轮开始后标记按新基准重新计算。
- 关闭并重开项目后仍能看到约定轮次的标记。
- 不依赖颜色也能区分“新建”和“更新”。

## 不在本任务范围内

- 展示删除占位。
- 文件事件时间线。
- 修改文件。

## 独立性与原子性检查

- **单一职责**：只计算并展示两个文件变化标记。
- **可独立验证**：连续执行两轮新建和修改操作即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
