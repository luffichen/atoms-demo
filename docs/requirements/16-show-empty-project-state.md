# 展示项目空状态

## 目标

当前游客没有项目时，用户能够理解当前状态并方便地开始创建第一个项目。

## 需求

1. 确认当前游客没有项目后，显示“暂无项目”。
2. 同时显示“创建第一个项目”按钮。
3. 点击按钮返回首页并聚焦需求输入框。
4. 点击按钮不会自动创建项目或自动提交任何内容。
5. 项目列表加载失败时不得显示该空状态。

## 验收标准

- 没有项目的游客看到空状态和创建按钮。
- 点击按钮后进入当前游客首页，输入框获得焦点。
- 未输入并提交需求前，项目列表仍为空。
- 加载失败场景显示错误而不是空状态。

## 不在本任务范围内

- 校验需求或创建项目。
- 项目列表加载错误。
- 首页其他内容。

## 独立性与原子性检查

- **单一职责**：只展示和响应项目为空的状态。
- **可独立验证**：使用没有项目的游客即可完整验证。
- **结论**：任务足够独立且原子，无需继续拆分。
