# 恢复项目实时连接

## 目标

项目页暂时断网时，用户不会误以为任务停止，也不会丢失草稿或断线期间的执行进度。

## 需求

1. 连接中断不停止服务端正在执行的任务和队列。
2. 页面显示“连接已断开，正在重连”。
3. 断线期间保留输入文字和待发送图片，但禁止发送。
4. 页面自动尝试重新连接。
5. 重连成功后补齐断线期间的消息和事件。
6. 补齐内容不得与断线前已有内容重复。
7. 恢复后继续接收实时更新并恢复发送能力。
8. 自动重连持续 60 秒仍失败时，显示“重新连接”按钮，同时继续后台尝试。
9. 用户可以返回“我的项目”，草稿仍保持。

## 验收标准

- 执行中断开网络，页面显示状态而任务继续。
- 断线期间不能发送，但文字和图片不丢失。
- 恢复网络后自动补齐所有缺失事件且没有重复。
- 连续失败 60 秒后出现手动入口，点击可以再次尝试。
- 离开并重开项目后可看到服务端最新状态。

## 不在本任务范围内

- 服务重启。
- 任务执行失败。
- 浏览器离线编辑。

## 独立性与原子性检查

- **单一职责**：只恢复页面与仍在运行服务之间的实时连接。
- **可独立验证**：在任务执行中断网、恢复并制造长时间失败即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
