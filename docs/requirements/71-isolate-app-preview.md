# 隔离生成应用预览

## 目标

生成应用无论正常、恶意或崩溃，都不能影响 Atoms Demo 主应用及其他项目。

## 需求

1. 生成网页不能读取或修改主应用的游客选择、项目状态、页面内容或浏览器存储。
2. 生成网页不能跳转或控制主应用页面。
3. 生成网页不能获得主应用凭据。
4. 预览崩溃不影响对话、任务队列、文件或终端。
5. 生成应用的网络失败只影响自身预览。
6. 外部站点不能替换当前预览。
7. 隔离不得阻止生成应用加载同一项目内的 CSS、JavaScript、图片和字体资源。

## 验收标准

- 生成应用尝试访问主应用状态或存储时无法成功。
- 生成应用无法把主页面导航到其他地址。
- 预览崩溃后其他项目功能继续可用。
- 外部链接和网络错误符合已确认的隔离行为。
- 使用独立 CSS 和 JavaScript 文件的生成应用在隔离 iframe 中仍能完整渲染和运行。

## 不在本任务范围内

- 应用预览正常导航。
- 智能体执行环境隔离。
- 用户身份认证。

## 独立性与原子性检查

- **单一职责**：只定义生成应用与主应用之间的运行边界。
- **可独立验证**：使用包含越界尝试和崩溃行为的测试应用即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
