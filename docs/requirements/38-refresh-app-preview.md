# 刷新应用预览

## 目标

用户在稳定时机看到最新生成结果，同时避免智能体修改中间状态造成频繁闪烁。

## 适用范围

- 仅适用于可在浏览器中运行的网页项目。

## 需求

1. 智能体执行期间不自动刷新应用预览。
2. 执行期间继续展示最近一次成功预览。
3. 每条任务完成后自动刷新一次应用预览。
4. 执行期间用户可以手动刷新，尝试查看当前结果。
5. 手动刷新不会改变任务执行状态。

## 验收标准

- 连续文件写入不会触发应用预览反复自动刷新。
- 任务完成后无需用户操作即可看到最新可用结果。
- 执行中手动刷新可以尝试当前结果，任务仍继续。
- 有历史成功预览时，执行期间始终有稳定内容可看。

## 不在本任务范围内

- 预览失败提示。
- 浏览器导航控件。
- 项目卡片缩略图。

## 独立性与原子性检查

- **单一职责**：只决定应用预览何时刷新。
- **可独立验证**：观察执行中、手动刷新和完成三个时机即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
