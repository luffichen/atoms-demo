# 提供默认游客

## 目标

不想创建自定义游客的用户，可以直接使用系统提供的默认游客体验全部首版功能。

## 需求

1. 系统始终存在一个名称为 `default` 的游客。
2. `default` 出现在游客选择列表中，并可以被选择和切换。
3. `default` 可以使用与其他游客相同的首版功能。
4. 用户不能再次创建名为 `default` 的游客。
5. 用户不能删除 `default` 游客。

## 验收标准

- 首次打开一个全新的演示实例时，游客列表中已有 `default`。
- 不创建任何自定义游客也能选择 `default` 并进入首页。
- 使用 `default` 可以创建、打开并继续修改项目。
- 尝试创建 `default` 或仅大小写不同的名称时不会产生新游客。
- 页面中不存在删除 `default` 的入口。

## 不在本任务范围内

- 创建其他游客。
- 记住或切换当前游客。
- 管理 `default` 下的项目。

## 独立性与原子性检查

- **单一职责**：只保证默认游客可用且不可重复创建或删除。
- **可独立验证**：无需创建其他游客即可覆盖全部验收场景。
- **结论**：任务足够独立且原子，无需继续拆分。
