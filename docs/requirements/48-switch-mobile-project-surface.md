# 切换窄屏项目区域

## 目标

窄屏用户一次专注于对话或工作区，同时能够发现另一侧的新活动。

## 需求

1. 项目页低于 1024px 时使用单区模式。
2. 顶部提供“对话 / 工作区”切换。
3. 进入项目时默认显示对话。
4. 文件或终端有新活动时，在“工作区”上显示提示。
5. 新活动不得自动抢占当前对话页面。
6. 进入工作区后，再选择“应用 / 文件 / 终端”。
7. 1024px 及以上恢复双区布局。

## 验收标准

- 在断点两侧分别出现单区和双区布局。
- 窄屏进入项目先看到对话。
- 新事件只增加工作区提示，当前页面不跳转。
- 用户可手动切到工作区并选择具体模式，再切回对话。

## 不在本任务范围内

- 一级导航抽屉。
- 工作区三个查看器内部行为。
- 桌面宽度拖动。

## 独立性与原子性检查

- **单一职责**：只控制窄屏项目页的两个主区域切换。
- **可独立验证**：在窄屏制造文件和终端事件即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
