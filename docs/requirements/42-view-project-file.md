# 查看项目文件

## 目标

用户可以只读查看常见项目文件，并明确知道无法预览的内容。

## 需求

1. 文本文件以带行号的只读内容展示。
2. 当前项目、实时写入和历史版本中的文本文件共用同一只读查看器。
3. TypeScript/TSX、JavaScript/JSX、JSON、CSS、HTML/XML、Markdown、Python、
   Go、Rust、Shell、YAML、TOML、SQL、Dockerfile、`.gitignore` 与 `.env`
   按类型进行语法高亮；HTML 内嵌 CSS 与 JavaScript 同时高亮。
4. 无扩展名文件只根据明确的 Python、Node.js 或 Shell shebang 识别语言，不扫描正文猜测。
5. 无法识别类型的文本文件按纯文本展示。
6. 查看器顶部固定显示完整相对路径、语言标签和必要的降级状态。
7. 高亮配色参考 GitHub Light，并服从产品现有背景、边框、焦点和字体 token。
8. 行号不可选中或点击；复制正文不包含行号。Tab 按 2 个空格宽度显示，但不改写原始字符。
9. 超过 200 KB 或 5,000 行时完整展示文本和行号，但关闭语法高亮并说明原因。
10. 实时写入的文本立即追加，高亮约 100 ms 防抖刷新，写入完成后立即完成最终高亮。
11. PNG 和 JPEG 图片直接预览。
12. 其他二进制文件显示名称、类型、大小和“无法预览”。
13. 超过 1 MB 的文本文件显示文件信息和“文件过大，无法预览”，不截断展示。
14. 切换文件时显示目标路径和加载状态；快速连续选择时只有最后一次请求可以更新内容。
15. 高亮失败时降级为完整纯文本，并显示“高亮不可用”，不阻断其他文件。
16. CRLF 与 LF 均正确分行；复制时统一为 LF；文件末尾换行不生成额外空白行号。
17. 首版不提供代码折叠、文件内搜索或跳转定义。
18. 代码长行不自动换行，通过横向滚动查看。
19. 首版不提供换行模式切换。
20. 首版不提供下载。
21. 首版不允许编辑、保存、新建、重命名、移动或删除文件。

## 验收标准

- 普通文本、PNG、JPEG、其他二进制和超大文本分别按约定展示。
- 文本内容有行号且不可修改。
- 当前文件、实时文件和历史版本文件使用一致的语言识别与高亮。
- 大型文本与高亮异常均按规则降级，内容不丢失。
- 文件加载不会被较早返回的请求覆盖。
- 超大文本不会展示不完整片段。
- 页面中没有下载或文件修改入口。

## 不在本任务范围内

- 文件树。
- 实时文件写入协议与传输。
- 文件变化标记。
- Diff 语法高亮。
- 主题切换、字号调节、行选择与行链接。
- 移动端文件树结构重做。

## 独立性与原子性检查

- **单一职责**：只负责呈现一个已选文件。
- **可独立验证**：准备四类文件即可覆盖。
- **结论**：任务足够独立且原子，无需继续拆分。
