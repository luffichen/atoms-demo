# 展示执行事件卡片

## 目标

用户可以在对话时间线中快速了解智能体执行过哪些文件操作和终端命令，并打开对应详情。

## 需求

1. 文件操作和终端命令分别显示为紧凑事件卡片。
2. 其他工具事件显示通用事件卡片。
3. 卡片展示事件类型、目标文件或命令、状态和时间。
4. 左侧卡片默认不完整展开文件内容或终端输出。
5. 点击文件事件卡片，在右侧打开对应文件查看器。
6. 点击终端事件卡片，在右侧打开对应终端查看器。
7. 通用事件没有右侧查看器，点击只展开安全摘要。
8. 当前选中的事件具有明确选中状态。
9. 每轮执行事件归属于该轮的 `luffi · 工程师` 消息组，智能体头像和名称显示在事件卡片之前。

## 验收标准

- 时间线可区分文件事件和终端事件。
- 每张卡片均包含约定摘要信息。
- 点击不同类型卡片时，右侧打开正确的查看器和对应内容。
- 左侧不会因完整输出而被大量占用。
- 文件或终端事件出现时，其上方能够看到 `luffi · 工程师` 身份，事件、思考状态和最终回复共享同一智能体消息组。

## 不在本任务范围内

- 文件或终端内容的流式更新。
- 查看器手动切换。
- 普通用户与智能体消息。

## 独立性与原子性检查

- **单一职责**：只负责事件摘要与详情入口。
- **可独立验证**：准备一个文件事件和一个终端事件即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
