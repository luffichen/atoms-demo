# 处理不可用项目

## 目标

用户尝试打开不存在、损坏或不属于当前游客的项目时，能够安全返回自己的项目列表。

## 需求

1. 项目不存在、无法读取或不属于当前游客时，不进入项目对话页。
2. 页面统一显示“项目不可用”，不透露项目是否属于其他游客或其具体信息。
3. 页面提供“返回我的项目”按钮。
4. 点击按钮返回当前游客的“我的项目”页面。
5. 错误不会自动切换游客。

## 验收标准

- 打开不存在的项目时显示“项目不可用”和返回按钮。
- 打开损坏项目时采用相同处理。
- 尝试打开其他游客的项目时，不显示其名称、对话、文件或其他信息。
- 点击返回后看到当前游客自己的项目列表。
- 全部错误场景均保持当前游客不变。

## 不在本任务范围内

- 修复损坏项目。
- 身份认证。
- 从正常项目卡片进入项目。
- 项目页内部错误。

## 独立性与原子性检查

- **单一职责**：只处理无法安全打开项目的入口错误。
- **可独立验证**：分别使用不存在、损坏和其他游客的项目即可验证。
- **结论**：三个原因对用户采用同一安全行为，任务无需继续拆分。
