# 跟随查看器最新输出

## 目标

用户可以自动查看文件或终端的最新流式内容，也可以暂停跟随以阅读较早内容。

## 需求

1. 文件查看器位于底部时，自动跟随新的写入内容。
2. 用户向上滚动后暂停，并显示“文件有新内容”。
3. 点击提示后回到最新位置并恢复跟随。
4. 终端查看器采用相同规则，暂停时显示“终端有新输出”。
5. 命令或写入结束不会强制改变用户当前阅读位置。

## 验收标准

- 两种查看器位于底部时均持续显示最新内容。
- 向上滚动后新增内容不改变阅读位置，并出现对应提示。
- 点击提示后回到底部并恢复自动跟随。
- 在暂停状态结束事件，阅读位置保持不变。

## 不在本任务范围内

- 对话时间线滚动。
- 文件和终端内容本身的流式传输。
- 手动打开其他查看器。

## 独立性与原子性检查

- **单一职责**：只控制流式查看器是否跟随最新输出。
- **可独立验证**：文件与终端遵循同一行为模型，可共同验证。
- **结论**：任务集中且无需继续拆分。
