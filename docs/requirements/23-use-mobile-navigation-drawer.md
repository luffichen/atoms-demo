# 使用窄屏导航抽屉

## 目标

屏幕较窄时，用户可以临时打开导航，而主内容不会被固定侧边栏持续挤压。

## 需求

1. 屏幕宽度低于 1024px 时，不固定展示桌面侧边栏或图标栏。
2. 页面顶部提供菜单按钮。
3. 点击菜单按钮后，以覆盖主内容的抽屉展示完整导航。
4. 抽屉打开时包含当前游客、首页和“我的项目”。
5. 选择首页或“我的项目”后，抽屉自动关闭。
6. 用户可以不导航而直接关闭抽屉。
7. 项目对话页抽屉中的游客信息不可切换。

## 验收标准

- 在 1024px 以下宽度进入页面时，导航默认关闭且主内容占据可用宽度。
- 菜单按钮可以打开和关闭导航抽屉。
- 选择首页或“我的项目”后进入对应页面且抽屉关闭。
- 项目页打开抽屉时无法切换游客。
- 键盘操作时焦点不会进入被抽屉遮挡的页面内容。

## 不在本任务范围内

- 桌面侧边栏折叠。
- 首页、项目列表或项目页的响应式内容布局。
- 游客切换逻辑。

## 独立性与原子性检查

- **单一职责**：只定义窄屏下一级导航的打开、关闭和选择行为。
- **可独立验证**：在临界宽度上下操作导航即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
