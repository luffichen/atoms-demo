# 在项目页锁定游客

## 目标

用户进入具体项目后，项目对话与所属游客保持固定，避免在项目上下文中切换身份造成数据混淆。

## 需求

1. 项目对话页显示当前游客名称。
2. 游客名称在项目对话页仅作为不可交互的信息。
3. 项目对话页不展示游客下拉箭头、切换菜单或禁用状态的切换控件。
4. 用户必须先返回首页或“我的项目”，才能切换游客。
5. 项目页中的对话、文件、执行过程和预览始终属于进入该项目时的游客。

## 验收标准

- 从某游客首页打开项目后，项目页能看到该游客名称。
- 点击或操作游客名称不会打开游客列表。
- 项目页不存在其他游客的切换入口。
- 返回首页或“我的项目”后，游客切换入口重新可用。
- 查看和继续项目期间不会出现其他游客的对话或项目数据。

## 不在本任务范围内

- 首页切换游客。
- 返回首页的导航样式。
- 项目对话、文件或预览的具体展示。

## 独立性与原子性检查

- **单一职责**：只限制项目页中的游客上下文。
- **可独立验证**：进入任一项目并检查游客控件与数据归属即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
