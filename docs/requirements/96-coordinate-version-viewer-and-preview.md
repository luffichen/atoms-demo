# 协调版本查看器与应用预览

## 目标

版本流程、文件事件、终端事件和应用预览在同一工作台中保持清晰且不抢夺用户主动阅读位置。

## 需求

1. 存在活动结构化需求时，重新打开项目默认进入“版本 → 当前工作”。
2. 没有结构化需求时，可预览项目默认进入“应用”，否则进入“文件”。
3. 直接编码存在未发布改动时不强制进入“版本”，版本入口显示徽标。
4. 用户未手动干预时，开发与测试中的文件和终端事件沿用自动切换。
5. 用户手动切回“版本”后，本轮执行不再抢走查看器；继续或重试恢复自动跟随。
6. 结构化需求讨论和技术设计阶段保持在“版本”。
7. 需求讨论和技术设计阶段展示当前正式预览。
8. 开发阶段显示“开发预览 · Rxxx”，待上线显示“待上线预览 · Rxxx”，发布后显示“正式版本 · Vn”。
9. 直接编码改动显示“开发预览 · 未发布”，发布后切换为正式版本。
10. 放弃工作后恢复当前正式版本预览。

## 验收标准

- 各工作状态打开项目时进入正确查看器。
- 自动跟随不会打断用户主动阅读版本文档。
- 预览内容和状态标签始终对应当前工作或正式版本。

## 不在本任务范围内

- 外部生产部署。
- 历史版本应用预览。

## 独立性与原子性检查

- **单一职责**：只协调工作状态与现有查看器/预览选择。
- **可独立验证**：遍历两类工作项的阶段和手动切换行为即可验证。
- **结论**：任务足够独立且原子。
