# 保留演示数据

## 目标

游客可以长期重新打开已有项目和持续对话，不因产品自动清理而意外丢失。

## 需求

1. 首版不自动过期或清理游客、项目、对话和项目文件。
2. 产品界面不提供删除游客或项目的能力。
3. 管理员可以在产品之外手动清理演示数据。
4. 被管理员清理的项目再次访问时，使用统一“项目不可用”流程。
5. 清理其他项目不得影响仍然存在的游客和项目。

## 验收标准

- 跨越多次访问后，未被管理员清理的数据仍可恢复。
- 页面中不存在删除游客或项目入口。
- 管理员清理项目后，旧入口显示“项目不可用”。
- 未清理的其他项目仍可正常打开。

## 不在本任务范围内

- 管理员清理工具或界面。
- 数据备份和恢复。
- 自动配额回收。

## 独立性与原子性检查

- **单一职责**：只定义首版数据的保留与外部清理边界。
- **可独立验证**：跨重启访问并模拟外部清理即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
