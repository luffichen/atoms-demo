# 保护版本内容边界

## 目标

正式代码版本只包含安全、可复现且属于当前项目的文件。

## 需求

1. 版本包含源码、配置、静态资源、锁文件、文档及必要构建/部署配置。
2. 强制排除 Git 元数据、session、智能体元数据、依赖、缓存、临时文件、日志、构建产物、运行时状态、本地数据库和秘密。
3. 平台维护不可被项目 `.gitignore` 取消的安全排除规则；项目可以追加普通规则。
4. 单个进入 Git 的二进制文件不超过 10 MB，首版不提供 Git LFS。
5. 拒绝 `.gitmodules`、submodule gitlink、嵌套 `.git`、项目外符号链接、设备文件和 Socket。
6. 内部安全符号链接只有在目标仍位于项目边界内时才允许。
7. 版本发布前重复执行边界与敏感文件检查。
8. 历史文件读取和 Diff 只访问受控 Git 对象，不接受任意 Git revision 或项目外路径。

## 验收标准

- 被强制排除或越界的内容无法进入检查点和正式版本。
- 项目 `.gitignore` 无法覆盖平台安全规则。
- 10 MB 以上二进制、submodule 和逃逸符号链接阻止发布并返回明确错误。

## 不在本任务范围内

- 大文件托管。
- 密钥管理。
- 远程 Git。

## 独立性与原子性检查

- **单一职责**：只负责版本收集、Git 对象访问和安全边界。
- **可独立验证**：构造各类排除、超大、嵌套和越界文件即可验证。
- **结论**：任务足够独立且原子。
