# 浏览工作记录

## 目标

用户可以查看所有结构化需求和直接编码工作项，包括未形成正式版本的已放弃工作。

## 需求

1. “工作记录”列出工作项类型、标题、需求编号（如有）、状态、版本号（如有）、创建和结束时间。
2. 已上线工作项链接对应正式版本。
3. 历史工作项详情提供只读对话和对应终端记录。
4. 历史对话左侧显示归档标识和“返回当前工作/开始新工作”入口。
5. 已放弃结构化需求展示草稿文档、放弃原因、最后快照文件树和相对基线 Diff。
6. 已放弃直接编码工作只展示对话、终端和放弃元数据，不展示已清除代码。
7. 工作记录和 session 不允许删除或重命名。
8. 版本、工作项、历史对话和当前工作均支持可复制深链接。

## 验收标准

- 发布和放弃的两类工作项均可追溯。
- 历史对话不能发送消息。
- 深链接刷新后恢复相同记录和查看状态。

## 不在本任务范围内

- 恢复或继续已放弃工作。
- 删除工作记录。
- 搜索与筛选。

## 独立性与原子性检查

- **单一职责**：只负责历史工作项记录与归档内容访问。
- **可独立验证**：准备四种“类型 × 终态”组合即可验证。
- **结论**：任务足够独立且原子。
