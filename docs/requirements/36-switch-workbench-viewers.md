# 切换右侧查看器

## 目标

用户可以在应用预览、项目文件和终端输出之间切换，同时避免实时事件打断主动阅读。

## 需求

1. 可预览网页项目提供“应用”“文件”“终端”三个顶层模式。
2. 不能在浏览器中预览的项目隐藏“应用”，只提供“文件”和“终端”。
3. 项目后续具备网页预览能力时，再显示“应用”。
4. 文件事件发生时，默认自动打开“文件”模式并选中事件的目标文件；命令事件发生时默认自动打开“终端”模式。
5. 用户手动选择任一模式后，本轮任务内不再强制切换。
   - 手动选择“文件”模式只锁定顶层模式，不阻止文件写入事件自动打开目标文件。
   - 用户在文件树中明确选择具体文件后，才锁定文件阅读位置。
6. 其他模式有更新时显示新活动提示。
7. 下一轮用户消息开始执行时同步恢复事件驱动的自动切换，首个工具事件不能读到上一轮遗留的锁定状态。

## 验收标准

- 未手动干预时，文件写入和终端命令分别打开正确模式。
- 用户手动切换后，新事件不会抢走当前模式。
- 有更新的其他模式出现可见提示。
- 下一轮开始后，自动切换重新生效。

## 不在本任务范围内

- 各查看器内部内容。
- 重新打开项目时的默认模式。
- 窄屏工作区布局。

## 独立性与原子性检查

- **单一职责**：只控制三个顶层模式的选择权。
- **可独立验证**：制造事件并插入一次手动切换即可覆盖。
- **结论**：任务足够独立且原子，无需继续拆分。
