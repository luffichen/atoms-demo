# 布局项目卡片网格

## 目标

“我的项目”在受支持的桌面窗口宽度下清晰排列卡片，不压缩内容或产生无意义留白。

## 需求

1. 768–1023px 宽的桌面 Chrome 窗口使用两列卡片。
2. 1024px 及以上使用三列卡片。
3. 卡片区域最大内容宽度为 1200px，并在更宽窗口中保持居中。
4. 列间距和卡片尺寸遵循项目 `atoms-design-style`。
5. 加载更多项目后继续使用同一网格。

## 验收标准

- 在 768、1023、1024 及更宽窗口检查列数符合规则。
- 卡片内容不因列数变化发生遮挡或不可读截断。
- 超宽窗口中内容不会无限拉宽。
- 加载更多后卡片顺序和网格连续。

## 不在本任务范围内

- 项目卡片内部信息。
- 项目列表数据加载。
- 小于 768px 的布局验收。

## 独立性与原子性检查

- **单一职责**：只负责项目卡片容器的响应式排列。
- **可独立验证**：调整桌面窗口到边界宽度即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
