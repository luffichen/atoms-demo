# 完成智能体消息

## 目标

用户能够明确知道一轮智能体任务何时真正结束，并获得简洁可行动的结果说明。

## 需求

1. 智能体已给出面向用户的最终回复，且该轮全部文件和终端操作均结束后，消息才标记“已完成”。
2. 应用预览结果独立，不影响消息完成状态。
3. 最终回复简要说明本轮完成了什么和主要变化。
4. 最终回复说明未完成事项或已知问题；没有时可明确说明。
5. 最终回复不重复粘贴完整代码或终端输出。
6. 消息完成后自动推进下一条排队消息。

## 验收标准

- 仍有文件或命令事件运行时，消息不会提前显示完成。
- 操作全部结束但尚无最终回复时，消息仍不显示完成。
- 预览失败但智能体工作正常结束时，消息可以完成。
- 最终回复包含结果摘要与问题说明，详情仍在右侧查看。
- 完成后队列正确推进。

## 不在本任务范围内

- 预览失败处理。
- 执行失败或超时。
- 最终回复的具体文风。

## 独立性与原子性检查

- **单一职责**：只建立一轮消息的正常完成条件和结果说明。
- **可独立验证**：控制操作、回复和预览的完成顺序即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
