# 初始化项目代码仓库

## 目标

每个新项目从创建起拥有由平台托管的独立 Git 仓库，为后续工作项检查点和正式代码版本提供稳定基线。

## 需求

1. 创建项目时初始化独立 Git 仓库和正式分支 `main`。
2. 初始化提交只包含平台要求的基础版本控制配置，不显示为正式版本。
3. 第一个用户可见版本从 `V1` 开始。
4. Git 元数据位于智能体不可写、文件查看器不可见的项目管理目录。
5. 平台独占分支、commit、merge 和 tag 的写权限。
6. 首版不连接 GitHub、GitLab 等远程仓库，不支持 submodule、嵌套仓库或 Git bundle。
7. 功能上线前清空现有项目数据，不实现旧项目和旧会话迁移。

## 验收标准

- 新项目创建后存在可用的 `main` 基线，但版本历史为空。
- 智能体无法直接修改 Git 历史或版本控制元数据。
- 第一次正式发布生成 `V1`，而不是 `V2`。

## 不在本任务范围内

- 正式版本发布。
- 远程仓库同步。
- 历史数据迁移。

## 独立性与原子性检查

- **单一职责**：只负责建立安全的项目级版本仓库基线。
- **可独立验证**：创建项目并检查仓库、权限和空版本历史即可验证。
- **结论**：任务足够独立且原子。
