# 展示代码版本历史

## 目标

用户可以在项目维度快速了解所有已发布代码版本及当前正式版本。

## 需求

1. 查看器顶部新增与“应用、文件、终端”并排的“版本”入口。
2. 内部次级页面为“当前工作、版本历史、工作记录”。
3. 版本历史只包含正式发布的代码版本，不混入检查点或已放弃需求。
4. 按版本号倒序展示，当前正式版本有明确标识。
5. 每项展示版本号、来源类型、需求编号（如有）、标题、上线时间和变化摘要。
6. 首次加载 20 条，支持“加载更多”。
7. 首版不提供搜索、筛选、排序切换或分组。
8. 没有版本时显示引导完成当前工作或开始新工作的空状态。
9. 项目列表卡片展示当前版本、活动需求阶段或未发布改动状态。

## 验收标准

- 正式版本顺序、当前标识和来源准确。
- 内部检查点和已放弃记录不会出现在版本列表。
- 分页和空状态正确。

## 不在本任务范围内

- 版本详情。
- 工作记录详情。
- 应用版本列表。

## 独立性与原子性检查

- **单一职责**：只负责正式代码版本列表与项目级摘要。
- **可独立验证**：准备多种来源和超过 20 个版本即可验证。
- **结论**：任务足够独立且原子。
