# 同步与恢复版本工作流

## 目标

多标签页、服务重启和执行失败不会导致重复流转、状态分叉或自动误恢复。

## 需求

1. 工作项问题、流程状态和文档带服务端修订号。
2. 多标签页通过实时事件同步对话、文档、阶段、执行状态和版本结果。
3. 阶段操作使用幂等键，同一确认最多成功一次。
4. 过期状态上的回答或动作被拒绝并刷新最新状态。
5. 服务重启、超时、智能体失败或用户停止后停留在当前阶段，记录原因，不自动继续。
6. 用户通过“继续执行”或“重试”手动恢复，无需二次确认。
7. 恢复前智能体检查当前工作区、Diff、待办和最近检查点。
8. 无法安全判断时停止并一次询问一个具体问题。
9. 发布事务启动后可在刷新页面后继续查看进度和最终结果。

## 验收标准

- 多标签同时确认只产生一次状态变化或版本。
- 重启后阶段和现场保持一致，且不会自动执行。
- 重试从现有现场继续，不清除用户代码。

## 不在本任务范围内

- 多活动工作项。
- 自动恢复。
- 跨实例分布式执行。

## 独立性与原子性检查

- **单一职责**：只负责工作流一致性、同步和人工恢复。
- **可独立验证**：覆盖多标签竞态、重启、超时和手动停止即可验证。
- **结论**：任务足够独立且原子。
