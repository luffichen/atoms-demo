# 选择项目默认查看器

## 目标

重新打开项目时，系统根据项目当前状态展示最相关的右侧内容。

## 需求

1. 空闲项目已有成功预览时默认打开“应用”。
2. 空闲网页项目从未有可用预览时默认打开“文件”。
3. 不能在浏览器中预览的项目默认打开“文件”。
4. 执行中项目正在运行命令时默认打开“终端”。
5. 执行中项目正在写文件时默认打开“文件”。
6. 执行中项目暂无文件或终端事件时，有成功预览打开“应用”，否则打开“文件”。
7. 首版不恢复上次关闭时的手动查看器选择。

## 验收标准

- 逐一准备上述项目状态，打开后均进入约定模式。
- 关闭前手动选择其他模式，重开后仍按项目状态而不是旧选择决定。

## 不在本任务范围内

- 打开项目入口。
- 事件到达后的自动切换。
- 各模式内部内容。

## 独立性与原子性检查

- **单一职责**：只在进入项目时选择一次默认模式。
- **可独立验证**：使用不同项目状态即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
