# 展示智能体思考状态

## 目标

智能体暂时没有产生用户可见文字或执行事件时，用户能够确认任务仍在继续，而不会误以为页面卡住。

## 需求

1. 收到当前运行消息的 `thinking` 状态后，页面展示“正在思考中...”。
2. 思考状态只传输是否活跃和所属消息，不传输或展示内部思考内容。
3. 用户可见回复文字开始输出、工具开始执行或消息进入终态时，思考提示立即消失。
4. 同一消息连续收到多个思考分片时，只展示一个思考提示。
5. 运行消息尚无用户可见回复和执行事件时，页面使用同一提示覆盖首次状态事件到达前的等待阶段。
6. 提示文字使用从左向右掠过的光斑动画表达持续活动；系统开启“减少动态效果”时改为静态文字。

## 验收标准

- 运行消息收到思考事件后显示且只显示一个“正在思考中...”。
- 事件携带的内部思考文字不会出现在页面或思考状态实时消息中。
- 收到首段回复、首个工具执行事件或完成、失败、取消状态后，提示消失。
- 光斑持续从提示文字左侧扫向右侧，不影响文字辨认。
- 开启“减少动态效果”后没有循环扫光，提示仍清晰可见。

## 不在本任务范围内

- 展示、存储或恢复模型思维链。
- 修改模型的思考级别或推理方式。
- 改变智能体回复、工具事件和消息终态的既有内容。

## 独立性与原子性检查

- **单一职责**：只反馈智能体处于思考等待阶段。
- **可独立验证**：模拟思考、正文、工具和终态事件即可验证完整状态变化。
- **结论**：任务足够独立且原子，无需继续拆分。
