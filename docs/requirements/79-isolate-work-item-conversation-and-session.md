# 隔离工作项对话与执行会话

## 目标

每个工作项拥有独立、可追溯的对话、智能体 session 和终端记录。

## 需求

1. 每个工作项创建全新 conversation，不继承其他工作项消息。
2. 每个工作项使用独立 session JSONL 目录，例如 `sessions/{projectId}/{workItemId}/`。
3. 同一结构化需求从访谈到上线持续使用同一工作项 session。
4. 同一直接编码工作项的多轮修改持续使用同一工作项 session。
5. 新工作项不得自动续接最近 session；需要历史信息时重新读取代码、文档和版本记录。
6. 终端事件按工作项隔离；新工作项从空终端开始。
7. 归档工作项的对话、终端和 session 随项目长期保留，但不进入 Git 或文件查看器。

## 验收标准

- 两个工作项的 JSONL、对话和终端事件互不混排。
- 同一工作项内多轮消息保持上下文连续。
- 历史工作项只读且可重新查看。

## 不在本任务范围内

- 对话内容呈现细节。
- 工作记录导航。
- 项目整体删除。

## 独立性与原子性检查

- **单一职责**：只负责工作项级上下文和执行记录隔离。
- **可独立验证**：创建两个工作项并比较 session、对话和终端边界即可验证。
- **结论**：任务足够独立且原子。
