# 流式展示智能体回复

## 目标

用户无需等待整段回复结束即可持续看到 `luffi` 面向用户的文字进展。

## 需求

1. 智能体用户可见文字以流式方式原位追加。
2. 同一段回复的分片不得生成多条重复消息。
3. 网络重连后补齐缺失文字，不重复已有部分。
4. 内部推理、隐藏提示和系统上下文始终不展示。
5. 流式回复遵循 1 秒实时事件延迟目标。

## 验收标准

- 多分片回复在同一消息中连续增长。
- 断线恢复后文本完整且没有重复段落。
- 页面中无法看到内部推理或隐藏上下文。
- 正常网络下 95% 文本分片在产生后 1 秒内可见。

## 不在本任务范围内

- Markdown 格式渲染。
- 文件和终端事件。
- 最终完成条件。

## 独立性与原子性检查

- **单一职责**：只传递和拼接用户可见文字流。
- **可独立验证**：使用多分片并在中途断网即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
