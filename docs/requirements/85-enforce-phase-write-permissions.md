# 限制各阶段写入权限

## 目标

平台从工具和文件层面执行阶段门禁，防止智能体绕过已确认流程。

## 需求

1. 需求讨论阶段可读代码、搜索代码并维护需求草稿，但不能修改业务代码。
2. 技术设计阶段可读代码并维护技术草稿，但不能提前实现。
3. 开发阶段开放项目代码写入、依赖安装、构建和测试工具。
4. 已确认需求文档和技术方案在开发、测试及待上线阶段只读。
5. 写入工具和终端命令均不得绕过文档锁。
6. 修改已确认需求必须先退回需求讨论；修改已确认技术方案必须先退回技术设计。
7. 测试通过进入待上线后冻结候选代码、技术方案和测试报告。
8. 平台管理目录、Git 元数据、其他工作项 session 和项目外路径始终不可写。

## 验收标准

- 各阶段允许与拒绝的文件操作符合矩阵。
- 终端命令不能绕过写入工具限制。
- 退回正确阶段后对应文档恢复可维护状态。

## 不在本任务范围内

- 阶段状态机交互。
- 具体开发任务。

## 独立性与原子性检查

- **单一职责**：只负责阶段与工具权限的强制映射。
- **可独立验证**：在各阶段尝试相同写操作即可验证。
- **结论**：任务足够独立且原子。
