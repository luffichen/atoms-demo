# 隔离不同标签页的当前游客

## 目标

同一浏览器同时使用多个标签页时，一个标签页切换游客不会让其他标签页的草稿或项目上下文突然改变。

## 需求

1. 每个已打开的标签页在本次使用期间保持自己的当前游客。
2. 一个标签页主动切换游客时，不自动改变其他已打开标签页的当前游客。
3. 每次主动选择或切换游客后，该游客成为浏览器最近记住的游客。
4. 之后新打开或重新加载的页面使用最近记住的游客。
5. 已经打开的项目页始终保持项目所属游客，不受其他标签页切换影响。

## 验收标准

- 标签页 A 和 B 最初均为游客甲时，在 A 切换到游客乙，B 仍保持游客甲。
- A 切换后，B 中尚未发送的草稿和项目内容不发生变化。
- A 切换到游客乙后新开标签页，新标签页直接进入游客乙首页。
- A 切换到游客乙后重新加载 B，B 使用最近记住的游客乙。
- 已打开的项目页不会因另一标签页切换游客而改变所属游客。

## 不在本任务范围内

- 首页游客切换的列表和失败行为。
- 跨标签页同步项目进度。
- 身份认证和权限控制。

## 独立性与原子性检查

- **单一职责**：只定义多个浏览器标签页之间的游客选择关系。
- **可独立验证**：同时打开两个标签页即可覆盖全部核心场景。
- **结论**：任务足够独立且原子，无需继续拆分。
