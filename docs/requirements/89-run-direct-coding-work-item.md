# 执行直接编码工作项

## 目标

用户无需需求文档和技术方案即可在独立工作项中多轮修改代码。

## 需求

1. 直接编码工作项从当前正式版本创建独立工作分支。
2. 首条消息后智能体可以立即读写代码、安装依赖和运行命令。
3. 同一工作项内支持现有消息队列和多轮跟进。
4. 每轮成功且受跟踪文件有变化时创建内部检查点。
5. 无文件变化的消息不创建检查点。
6. 失败或停止不创建完成检查点，保留现场供继续或重试。
7. 文件树变化标记相对工作项开始时的正式版本计算，而不是只看最近一轮。
8. 一个直接编码工作项最多发布一个正式版本；发布或放弃后对话与 session 归档。
9. 直接编码不创建需求编号、需求文档或技术方案。

## 验收标准

- 用户可以多轮编码而不进入结构化阶段。
- 成功轮次形成内部检查点，失败轮次不伪装为完成。
- 新工作项不会继承旧直接编码 session。

## 不在本任务范围内

- 正式版本验证与发布。
- 放弃未发布改动。

## 独立性与原子性检查

- **单一职责**：只负责直接编码工作项的执行和检查点。
- **可独立验证**：执行成功、无变化、失败和多轮修改即可验证。
- **结论**：任务足够独立且原子。
