# 选择模式开始新工作

## 目标

用户可以选择完整的需求研发流程，也可以保留立即编码的快捷体验。

## 需求

1. 首页创建项目时提供“直接编码”和“按需求研发”两种模式。
2. 默认选择“直接编码”，并在当前浏览器记住上次选择。
3. 项目没有活动工作项时，左侧提供空白新工作输入。
4. 直接发送内容创建直接编码工作项。
5. 输入框上方及“版本”的“当前工作”区域提供“按需求研发”入口。
6. 用户可通过“新建需求：……”等自然语言发起结构化需求创建。
7. 自然语言或按钮创建结构化需求均展示二次确认，确认后才创建工作项、分支、对话和 session。
8. 历史对话中的输入框保持只读，不能把新消息追加到已归档工作项。

## 验收标准

- 新项目默认进入直接编码，但用户可显式选择需求研发。
- 归档一项工作后发送新内容会创建全新工作项。
- 自然语言新建需求在确认前不改变项目状态。

## 不在本任务范围内

- 两种模式的内部执行流程。
- 工作项并发限制。

## 独立性与原子性检查

- **单一职责**：只负责选择模式并启动新工作。
- **可独立验证**：分别从首页、项目输入区和自然语言入口创建工作即可验证。
- **结论**：任务足够独立且原子。
