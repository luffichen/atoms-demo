# 同步同一项目的多个标签页

## 目标

同一项目同时在多个桌面 Chrome 标签页打开时，所有标签页看到一致的已发送内容和执行状态。

## 需求

1. 多个标签页实时同步已发送消息、事件、文件和任务状态。
2. 不同标签页提交的新消息按服务端收到顺序进入同一项目队列。
3. 队列容量由所有标签页共享。
4. 一个标签页取消排队消息或停止执行后，其他标签页立即同步结果。
5. 同一取消或停止操作不得重复执行。

## 验收标准

- 两个标签页打开同一项目时，任一标签页的新消息和事件在另一页出现。
- 两页近同时发送消息时只进入一条共享队列，顺序明确。
- 队列满状态在全部标签页一致。
- 取消或停止只执行一次，所有页面得到相同终态。

## 不在本任务范围内

- 同步未发送草稿。
- 不同项目之间的同步。
- 不同游客之间的数据共享。

## 独立性与原子性检查

- **单一职责**：只同步同一项目已提交的共享状态。
- **可独立验证**：使用两个标签页提交、取消和停止即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
