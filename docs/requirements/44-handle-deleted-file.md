# 处理文件删除

## 目标

智能体删除文件后，文件树和正在查看的内容能及时反映事实，同时保留操作上下文。

## 需求

1. 删除后的文件立即从文件树移除。
2. 文件树不保留“已删除”占位项。
3. 删除动作继续作为左侧事件，展示文件路径和结果。
4. 用户正在查看被删除文件时，内容区停止更新并显示“文件已删除”。
5. 删除提示继续显示该文件路径。
6. 不自动跳转到其他文件。

## 验收标准

- 删除后树中不再出现该文件，但时间线仍有删除事件。
- 正在查看该文件时显示明确删除提示和原路径。
- 内容区不再接收后续内容，也不会擅自选中其他文件。
- 用户选择其他文件后正常查看。

## 不在本任务范围内

- 恢复删除文件。
- “已删除”状态标记。
- 用户主动删除文件。

## 独立性与原子性检查

- **单一职责**：只同步一个文件被删除后的界面状态。
- **可独立验证**：分别删除未选中和已选中文件即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
