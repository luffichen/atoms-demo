# 满足实时事件延迟目标

## 目标

智能体执行过程中，用户能够近实时看到状态、文件和终端变化。

## 需求

1. 正常网络下，95% 的消息状态事件在服务端产生后 1 秒内可见。
2. 正常网络下，95% 的文件写入事件在服务端产生后 1 秒内可见。
3. 正常网络下，95% 的终端输出事件在服务端产生后 1 秒内可见。
4. 智能体用户可见文字分片遵循同一目标。
5. 指标不包含模型思考、命令执行或文件操作本身耗时。

## 验收标准

- 对四类实时内容分别采样并统计，至少 95% 满足 1 秒目标。
- 慢模型或长命令不会被错误计算为界面事件延迟。
- 持续流式输出期间页面无需刷新。

## 不在本任务范围内

- 普通页面首屏加载。
- 模型响应速度。
- 预览启动时间。

## 独立性与原子性检查

- **单一职责**：只验证服务端事件到页面呈现的延迟。
- **可独立验证**：使用可控事件时间戳即可测量。
- **结论**：任务足够独立且原子，无需继续拆分。
