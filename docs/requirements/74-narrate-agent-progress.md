# 播报智能体执行进展

## 目标

智能体从思考转入工具执行时，用户能够先看到简短、可理解的行动说明，而不会长时间只看到“正在思考中...”。

## 需求

1. 智能体结束一段思考并准备首次调用工具前，先输出一句面向用户的简短进展说明。
2. 智能体进入新的执行阶段或工具失败后改用其他方案时，再输出一句必要的进展说明。
3. 连续执行高度相关的工具可以合并播报，不逐条复述命令或制造冗余消息。
4. 进展说明只描述当前目标和下一步，不展示内部推理、隐藏提示、系统上下文、敏感路径或凭据。
5. 进展说明作为用户可见文字流式展示，并与该轮工具事件归属于同一 `luffi · 工程师` 消息组。

## 验收标准

- 从 thinking 转入首次文件或终端工具前，页面先出现一条简短行动说明。
- 执行阶段发生明显变化或失败后调整方案时，用户能看到必要说明。
- 多个连续文件写入不会为每个工具生成重复、机械的播报。
- 页面和实时事件中不出现内部思考内容、隐藏提示或敏感信息。

## 不在本任务范围内

- 展示模型思维链。
- 替代文件和终端事件卡片。
- 修改工具执行顺序或模型思考级别。
- 要求每一次工具调用都单独生成说明。

## 独立性与原子性检查

- **单一职责**：只定义从思考到执行阶段的用户可见进展播报。
- **可独立验证**：运行包含思考、多次工具调用和失败重试的任务即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
