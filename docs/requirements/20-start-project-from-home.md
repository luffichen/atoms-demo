# 从首页启动新项目

## 目标

用户提交一条有效需求后，系统创建一个完整可继续的项目并立即启动真实生成任务。

## 前置条件

- 当前游客有效。
- 文字需求和图片附件均已通过校验。

## 需求

1. 用户可以点击发送按钮或按 `Enter` 提交需求。
2. `Shift + Enter` 只插入换行，不提交。
3. 提交开始后，输入区显示“正在创建项目”，并阻止再次提交。
4. 每次成功提交只创建一个新项目和该项目唯一的持续对话。
5. 第一条文字和图片作为该对话的首条用户需求。
6. 项目创建成功后立即启动真实智能体生成任务，并进入项目对话页。
7. 项目自动命名由独立的命名任务负责；名称失败不影响项目创建。
8. 项目创建或生成任务启动失败时，明确提示失败。
9. 失败时不在项目列表留下没有可用对话的残缺项目。
10. 失败后恢复提交前的文字和图片，并允许用户重试。

## 验收标准

- 点击发送或按 `Enter` 均只创建一个项目并进入其对话页。
- 按 `Shift + Enter` 增加换行且不创建项目。
- 连续点击或重复按键不会创建多个项目。
- 成功后，项目首条消息与首页提交的文字和图片一致，真实生成任务已经开始。
- 项目创建失败或任务启动失败时，用户留在首页，看到错误，原草稿完整保留。
- 失败后“我的项目”中没有残缺项目；再次提交可以成功。

## 不在本任务范围内

- 校验文字和图片。
- 生成项目名称的具体规则。
- 智能体执行过程和项目页展示。
- 项目完成条件。

## 独立性与原子性检查

- **单一用户结果**：一次首页提交要么产生一个已启动且可继续的完整项目，要么不产生项目。
- **不可继续拆分的原因**：若把创建项目和启动首个任务拆开，会产生需求明确禁止的残缺中间状态。
- **可独立验证**：通过成功、重复提交、创建失败和启动失败四类场景即可验证完整事务边界。
- **结论**：任务作为一个原子用户事务保留，无需继续拆分。
