# 实时更新文件树

## 目标

智能体改变项目结构时，文件树及时反映最新事实，并尽量保持用户当前阅读上下文。

## 需求

1. 新建、重命名或移动文件的事件可见后 1 秒内更新文件树。
2. 更新时尽量保持现有目录展开状态。
3. 未受影响的当前选中文件保持选中。
4. 已选文件被重命名或移动时继续显示内容，并更新路径。
5. 不要求用户手动刷新文件树。

## 验收标准

- 三类文件结构操作均在目标时间内出现在树中。
- 无关目录不会因更新全部折叠。
- 已选文件移动后选择和内容保持，路径变化正确。
- 页面没有文件树刷新按钮依赖。

## 不在本任务范围内

- 文件内容流式写入。
- 文件变化状态标记。
- 文件删除后的特殊提示。

## 独立性与原子性检查

- **单一职责**：只同步非删除类文件结构变化。
- **可独立验证**：依次新建、重命名和移动文件即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
