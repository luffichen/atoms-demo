# 创建项目工作项

## 目标

用户每次开始一项新工作时获得独立的工作项、对话和执行身份，不与其他需求或编码任务混用上下文。

## 需求

1. 项目下的工作统一建模为“工作项”。
2. 工作项类型包括“结构化需求”和“直接编码”。
3. 每个工作项拥有不可变 ID、标题、类型、状态、创建时间和归档时间。
4. 结构化需求额外分配项目内递增编号 `R001、R002…`。
5. 已放弃的需求编号不复用。
6. 活动期间允许修改标题；发布或放弃后标题锁定。
7. 工作项发布或放弃后归档；下一项工作必须创建新的工作项。

## 验收标准

- 两次新工作拥有不同的工作项 ID。
- 结构化需求按顺序获得稳定需求编号。
- 归档工作项不能继续接收消息或修改标题。

## 不在本任务范围内

- 工作项内部阶段。
- 正式代码版本。
- 工作记录页面。

## 独立性与原子性检查

- **单一职责**：只负责创建和标识项目工作项。
- **可独立验证**：连续创建、归档并重新创建工作项即可验证。
- **结论**：任务足够独立且原子。
