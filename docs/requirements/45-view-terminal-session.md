# 查看终端会话

## 目标

用户可以在项目唯一的连续终端会话中理解各轮智能体任务执行的全部命令、输出和结果。

## 需求

1. 每个项目始终只有一个终端会话。
2. 顶部直接显示“终端”，不显示“会话 1”等编号标签。
3. 项目所有轮次的实际执行命令和输出按发生顺序连续追加。
4. 每条命令以 `$ 命令` 开始，后面紧跟其输出和最终结果。
5. 点击左侧某条命令事件时，打开对应终端会话并定位到该命令。
6. 终端会话持续更新，不为任务轮次或单条命令创建其他会话或独立详情页。
7. 终端只读，不允许输入命令或与程序交互。
8. 视觉组织参考 `docs/source/assets/terminal-session-reference.png`。

## 验收标准

- 跨越多轮任务执行多条命令时，均按实际顺序出现在同一个会话中。
- 每条命令与自己的输出相邻，能够区分成功、失败或取消结果。
- 点击任一命令事件后定位到会话中的正确位置。
- 页面中不存在终端输入入口。

## 不在本任务范围内

- 命令输出的流式接收。
- 通过对话发起命令。

## 独立性与原子性检查

- **单一职责**：只负责呈现一个项目的连续终端记录。
- **可独立验证**：跨多轮执行成功、失败或取消命令即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
