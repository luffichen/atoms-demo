# 处理应用预览失败

## 目标

网页无法预览时，用户能区分智能体任务结果与预览结果，并继续查看文件或重试。

## 适用范围

- 仅适用于可在浏览器中运行的网页项目。

## 需求

1. 任务已完成但预览失败时，任务状态仍保持“已完成”。
2. “应用”模式显示“预览失败”、简要原因和“重试预览”。
3. 文件模式继续可用。
4. 项目卡片保留最近一次成功预览。
5. 没有任何历史成功预览时，卡片显示“暂无预览”。
6. 重试只重新尝试预览，不重新执行用户消息。

## 验收标准

- 制造预览失败时，任务不被错误标为失败。
- 用户可以看到原因、重试，并切换到文件模式。
- 有历史预览和无历史预览的卡片分别符合约定。
- 点击重试不会新增对话任务或重复修改文件。

## 不在本任务范围内

- 智能体执行失败。
- 自动修复生成应用。
- 预览正常刷新时机。

## 独立性与原子性检查

- **单一职责**：只处理应用预览本身失败。
- **可独立验证**：使用有、无历史预览两种失败项目即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
