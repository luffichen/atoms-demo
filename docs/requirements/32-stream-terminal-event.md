# 实时展示终端命令

## 目标

智能体执行命令时，用户能够持续看到同一命令的输出和最终结果。

## 需求

1. 同一条终端命令在左侧只产生一张事件卡片。
2. 持续输出时，在原卡片上更新状态，不为输出分片新增事件。
3. 右侧终端查看器实时追加命令输出。
4. 命令结束后，卡片显示“成功”“失败”或“已取消”。
5. 卡片保留命令的退出结果。
6. 已显示输出在失败或取消后继续可见。
7. 单条命令最多展示 5 MB 输出，超过后停止追加并标记“输出已截断”。
8. 输出截断不停止命令，最终结果仍正常显示。
9. 工作区标签栏右侧的统一“自动跟随”开关开启时，新增命令及输出自动打开终端并滚动到底部。
10. 用户在终端中向上滚动超过 80px 时关闭统一自动跟随，后续输出不抢占阅读位置并提示“有新输出”。
11. 用户重新开启自动跟随后，如果终端是最近发生实时活动的查看器，立即切换到终端并滚动到最新输出。

## 验收标准

- 一条产生多段输出的命令始终只有一张事件卡片。
- 每段输出无需刷新即可在终端查看器出现。
- 成功、失败和取消分别显示正确终态和退出结果。
- 命令结束后可以继续查看完整已接收输出。
- 自动跟随开启时终端保持在最新输出；上滚后关闭，手动重新开启后恢复。

## 不在本任务范围内

- 用户在终端中输入命令。
- 多条命令的选择与切换。

## 独立性与原子性检查

- **单一职责**：只呈现一条命令从开始到终态的实时输出。
- **可独立验证**：分别执行成功、失败和取消的流式命令即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
