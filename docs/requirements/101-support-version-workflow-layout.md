# 适配版本工作流布局与性能

## 目标

版本管理在桌面和移动端保持可用、可访问，并在普通项目规模下及时响应。

## 需求

1. 桌面端“版本”查看器沿用紧凑工作台布局，不新增全局设置页。
2. 当前工作详情包含概览、需求、技术方案、变更、测试和发布标签；未进入阶段显示锁定说明。
3. 流程主操作只出现在输入框上方，不在右侧重复。
4. 移动端使用单列布局，流程栏允许两行，Diff 使用统一视图。
5. 触控目标至少 44px，键盘焦点、语义标签、错误说明和状态变化满足现有可访问性规范。
6. 当前工作、版本历史和工作记录首屏 p95 小于 2 秒。
7. 历史文件树 p95 小于 2 秒，单文件 Diff p95 小于 3 秒。
8. 验证、合并和发布不设短时限，但持续显示进度并允许刷新恢复。
9. 版本列表、工作记录、文档、Diff 均实现加载、空、错误和禁用状态。

## 验收标准

- 390px、768px、1024px 和 1440px 宽度下核心流程可完成且无页面级横向溢出。
- 键盘可完成查看、发起操作和二次确认。
- 性能样本满足目标，长任务持续反馈状态。

## 不在本任务范围内

- 搜索与高级筛选。
- 并排移动端 Diff。
- 应用版本界面。

## 独立性与原子性检查

- **单一职责**：只负责版本域的布局、可访问性和性能门槛。
- **可独立验证**：使用固定视口、键盘和性能样本即可验证。
- **结论**：这些横切质量属性共同约束同一版本工作台交付。
