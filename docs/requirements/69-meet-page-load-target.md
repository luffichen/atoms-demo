# 满足普通页面首屏目标

## 目标

用户在正常网络下能够快速进入游客选择、首页和项目列表的可交互状态。

## 需求

1. 正常网络下，95% 的游客选择页在 2 秒内可交互。
2. 正常网络下，95% 的首页在 2 秒内可交互。
3. 正常网络下，95% 的“我的项目”首屏在 2 秒内可交互。
4. 应用预览和智能体启动不计入本指标。
5. 未完成加载时显示明确加载状态，不能把缺少数据误显示为空状态。

## 验收标准

- 对三个页面分别采样，至少 95% 满足 2 秒目标。
- 项目列表加载中不会短暂显示“暂无项目”。
- 预览和智能体启动使用其独立状态，不阻塞普通页面验收。

## 不在本任务范围内

- 实时事件延迟。
- 应用预览启动。
- 模型响应时间。

## 独立性与原子性检查

- **单一职责**：只验证三个普通页面的可交互首屏时间。
- **可独立验证**：可在固定网络条件下分别测量。
- **结论**：任务足够独立且原子，无需继续拆分。
