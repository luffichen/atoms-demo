# 管理标签页本地草稿

## 目标

每个标签页的未发送内容互不干扰，并具有简单明确的刷新丢弃规则。

## 需求

1. 同一项目不同标签页的文字和待发送图片草稿互不共享。
2. 一个标签页发送或清空草稿不影响其他标签页。
3. 正常页面内操作和网络自动重连保留当前草稿。
4. 手动刷新首页或项目页后，不恢复任何未发送文字或图片。
5. 关闭标签页后不恢复草稿。
6. 刷新或关闭前不显示草稿丢失警告。

## 验收标准

- 两个标签页分别输入不同内容，互相不可见。
- 一页发送后另一页草稿保持。
- 网络断开并自动恢复后草稿仍在。
- 刷新或关闭重开后草稿为空，且此前没有警告。

## 不在本任务范围内

- 首页切换游客时携带草稿。
- 已发送消息同步。
- 服务端消息队列。

## 独立性与原子性检查

- **单一职责**：只规定未发送草稿在标签页内的生命周期。
- **可独立验证**：通过双标签、断网、刷新和关闭即可覆盖。
- **结论**：任务足够独立且原子，无需继续拆分。
