# 收起桌面项目对话区

## 目标

桌面用户可以临时隐藏对话区，让应用、文件或终端工作区占满可用空间。

## 需求

1. 桌面对话区顶部提供收起按钮。
2. 收起后右侧工作区占用释放出的空间。
3. 工作区保留“展开对话”入口。
4. 收起期间任务、消息和事件继续更新。
5. 重新展开后恢复原对话内容和滚动位置。
6. 收起状态只属于当前项目本次打开过程，不跨项目记忆。

## 验收标准

- 收起和展开操作均可用，工作区宽度正确变化。
- 收起期间任务继续执行且消息没有丢失。
- 展开后回到收起前阅读位置，并能看到期间新增内容提示。
- 打开另一个项目时对话区按正常状态展示。

## 不在本任务范围内

- 侧边栏折叠。
- 拖动对话区宽度。
- 窄屏单区切换。

## 独立性与原子性检查

- **单一职责**：只隐藏和恢复当前项目的对话区。
- **可独立验证**：收起期间产生实时活动后再展开即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
