# 调整项目双区宽度

## 目标

桌面用户可以按当前任务需要调整对话区和工作区所占空间。

## 需求

1. 1024px 及以上时，项目页默认约 30% 展示对话、70% 展示工作区。
2. 两区之间提供可拖动分隔线。
3. 对话区最小宽度为 320px。
4. 工作区最小宽度为 480px。
5. 拖动不能让任一区域小于最小宽度。
6. 浏览器记住最近一次比例，并用于之后打开的项目。

## 验收标准

- 首次进入桌面项目页时比例约为 30% / 70%。
- 分隔线可向两侧拖动，两个最小宽度均被遵守。
- 调整后打开另一个项目或刷新，比例保持。
- 低于 1024px 时不显示双区拖动分隔线。

## 不在本任务范围内

- 完全收起对话区。
- 窄屏单区切换。
- 两区内部内容。

## 独立性与原子性检查

- **单一职责**：只调整桌面双区宽度。
- **可独立验证**：拖动到两侧边界并刷新即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
