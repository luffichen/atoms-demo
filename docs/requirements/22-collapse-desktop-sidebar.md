# 折叠桌面侧边栏

## 目标

用户可以收窄桌面侧边栏，为首页内容或项目工作区腾出更多空间，同时保留主要导航能力。

## 需求

1. 宽屏侧边栏顶部提供折叠按钮，视觉参考 `docs/source/assets/sidebar-collapse-reference.png`。
2. 点击后侧边栏收窄为图标栏，而不是完全隐藏。
3. 图标栏保留展开按钮、当前游客、首页和“我的项目”图标。
4. 首页和“我的项目”的游客图标可以打开游客列表；项目页的游客图标不可交互。
5. 每个仅图标入口在鼠标悬停或键盘聚焦时显示文字提示。
6. 再次点击展开按钮恢复完整侧边栏。
7. 浏览器记住最近的展开或折叠状态。
8. 页面跳转和刷新后保持该状态。

## 验收标准

- 展开和折叠按钮均能正确改变侧边栏宽度。
- 折叠后仍能进入首页和“我的项目”。
- 每个图标均有可见文字提示，并可使用键盘访问。
- 项目外页面和项目页的游客图标分别符合可切换与不可切换规则。
- 折叠后跳转或刷新仍保持折叠；展开状态同理。

## 不在本任务范围内

- 窄屏抽屉导航。
- 一级导航页面内容。
- 游客切换本身。

## 独立性与原子性检查

- **单一职责**：只处理桌面侧边栏在完整栏与图标栏之间切换。
- **可独立验证**：在各一级页面切换、刷新并操作图标即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
