# 查看代码版本详情

## 目标

用户可以理解某个正式版本交付了什么、为什么修改以及如何验证。

## 需求

1. 详情展示版本号、来源、需求编号、标题、状态、上线时间和确认游客。
2. 展示 Git tag、短 commit ID、基线版本和用户可感知变化。
3. 展示需求文档、原子需求索引、技术方案、技术决策、测试报告和发布记录；直接编码版本明确缺失哪些结构化文档。
4. 展示文件变化统计和逐文件 Diff。
5. Diff 默认比较前一个正式版本；`V1` 与初始化基线比较。
6. 支持新增、修改、删除和重命名状态。
7. 文本 Diff 超过 5,000 行或 1 MB 时只显示统计和截断提示。
8. 二进制文件只显示元数据变化；单个文件 Diff 按需加载且最多同时展开一个。
9. 提供“查看工作对话”和“基于此版本查看代码”。
10. 正式版本不能删除、重命名、编辑、恢复、下载或与任意版本比较。

## 验收标准

- 两种来源版本展示正确的文档完整度和关联工作项。
- Diff 基线、状态、懒加载与截断正确。
- 所有正式版本详情保持只读。

## 不在本任务范围内

- 任意版本比较。
- ZIP/Git bundle 导出。
- 恢复到历史版本。

## 独立性与原子性检查

- **单一职责**：只负责正式版本详情和相邻版本差异。
- **可独立验证**：准备文本、重命名、删除、二进制和超大 Diff 即可验证。
- **结论**：任务足够独立且原子。
