# 通知后台任务结果

## 目标

用户离开项目页后，仍能知道后台消息已经结束，并快速返回对应项目。

## 需求

1. 用户不在对应项目页时，消息完成、失败或取消后显示站内通知。
2. 通知包含项目名称和结果。
3. 点击通知打开对应项目。
4. 同一消息终态只产生一次通知。
5. 通知提供独立关闭按钮；关闭通知不应打开项目。
6. 用户未操作时，通知在 8 秒后自动收起，避免持续遮挡页面。
7. 首版不发送浏览器系统通知、邮件或其他外部通知。

## 验收标准

- 离开项目页后，三种终态分别出现正确站内通知。
- 点击通知打开正确项目，不因项目同名而打开错误项目。
- 点击关闭按钮后通知立即消失，且当前页面不发生跳转。
- 通知展示 8 秒后自动消失。
- 多标签页同步终态不会产生重复通知。
- 浏览器和外部渠道没有通知请求。

## 不在本任务范围内

- 项目页内状态展示。
- 通知历史中心。
- 外部通知渠道。

## 独立性与原子性检查

- **单一职责**：只告知一个后台消息进入终态。
- **可独立验证**：在项目外触发三类终态即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
