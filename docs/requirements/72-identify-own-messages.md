# 标识自己的对话消息

## 目标

用户在项目对话中能够立即识别由自己发送的消息，并与智能体回复清晰区分。

## 需求

1. 每条用户消息显示当前游客的头像。
2. 用户消息不显示“我”、游客名称或其他身份名称，避免头像右侧出现冗余标签。
3. 用户头像沿用参与者身份规则，由当前游客名称首字符生成。
4. 历史消息、刚发送的消息和实时更新后的消息使用相同身份呈现。
5. 智能体名称和头像保持不变。

## 验收标准

- 用户消息旁显示与当前游客一致的首字符头像。
- 不同游客进入各自项目时头像字符正确，头像右侧不显示“我”或游客名称。
- 同一轮中的用户消息、执行事件和智能体回复仍保持原有顺序与状态。
- 项目重新打开或收到实时更新后，用户身份呈现不发生变化。

## 不在本任务范围内

- 上传或编辑头像。
- 修改侧边栏和游客选择页的游客名称。
- 修改智能体身份。
- 调整消息内容、状态或实时传输规则。

## 独立性与原子性检查

- **单一职责**：只定义项目对话中当前用户消息的身份呈现。
- **可独立验证**：使用不同名称的游客分别打开项目即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
