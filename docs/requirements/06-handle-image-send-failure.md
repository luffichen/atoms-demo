# 处理图片发送失败

## 目标

图片无法随消息正常提交时，系统不得让智能体收到缺少部分图片的不完整需求。

## 需求

1. 发送时，只要任意一张图片处理失败，整条消息暂不发送。
2. 系统应明确指出失败的图片。
3. 用户输入的文字和全部待发送图片均继续保留。
4. 用户可以移除失败图片，或重试失败图片。
5. 只有全部保留图片均可正常提交后，整条消息才可以发送。

## 验收标准

- 多图消息中任意一张处理失败时，智能体不会收到文字或其中一部分图片。
- 失败后，输入框文字和所有图片仍然存在，失败图片具有明确提示。
- 移除失败图片后，可以重新发送剩余的完整消息。
- 重试成功后，可以发送包含原有文字和全部图片的完整消息。

## 不在本任务范围内

- 添加、校验、预览或移除普通待发送图片。
- 智能体如何理解图片。
- 消息发送后的对话展示。

## 独立性与原子性检查

- **单一职责**：只处理图片在消息提交阶段失败的情况。
- **可独立验证**：通过制造一张图片提交失败即可验证消息完整性保护。
- **结论**：任务足够独立且原子，无需继续拆分。
