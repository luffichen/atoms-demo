# 离开正在运行的项目

## 目标

用户可以从项目页返回项目列表，而不会意外中止生成任务或丢失队列。

## 需求

1. 项目页顶部提供返回操作。
2. 返回固定进入当前游客的“我的项目”，不依赖进入来源或浏览器历史。
3. 离开项目页不取消正在执行的任务。
4. 离开项目页不取消排队消息。
5. 项目任务和队列在后台继续推进。
6. 再次打开项目时，恢复最新对话、状态、事件和结果。

## 验收标准

- 从首页创建、列表打开或直接进入的项目，返回均到当前游客“我的项目”。
- 执行中返回后，任务继续产生进度直至结束。
- 有排队消息时，离开后仍按顺序执行。
- 再次打开后看到离开期间发生的完整最新状态。

## 不在本任务范围内

- 停止任务。
- 项目列表卡片状态。
- 多标签页实时同步。

## 独立性与原子性检查

- **单一职责**：只定义离开项目页时任务与导航的结果。
- **可独立验证**：在执行和排队状态分别返回、重开即可验证。
- **结论**：任务足够独立且原子，无需继续拆分。
