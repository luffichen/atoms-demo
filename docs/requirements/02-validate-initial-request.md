# 校验首页首次需求

## 目标

用户从首页创建项目前，系统应判断文字需求是否有效，避免创建无法执行或内容不受控的项目。

## 需求

1. 首页首次需求必须包含至少一个非空白文字。
2. 只包含空格、制表符或换行的内容视为无效。
3. 附件不能代替文字需求。
4. 单次提交的文字需求最多包含 10,000 个可见字符。
5. 输入无效或超长时，不得创建项目或启动生成任务。
6. 输入区应明确说明不能提交的原因。
7. 用户将内容修改为有效文字后，可以再次提交。

## 验收标准

- 输入为空时，无法提交并创建项目。
- 只输入空格或换行时，无法提交并创建项目。
- 只添加附件而不输入文字时，无法提交并创建项目。
- 输入 1 至 10,000 个可见字符时，可以提交。
- 输入超过 10,000 个可见字符时，无法提交，输入区明确提示内容过长。
- 超长内容缩减到限制以内后，可以再次提交。
- 每种校验失败场景均不会出现在项目列表中留下新项目。

## 不在本任务范围内

- 添加、移除或读取附件。
- 创建项目。
- 自动生成项目名称。
- 启动智能体任务。
- 对项目内后续消息进行校验。

## 独立性与原子性检查

- **单一职责**：只判断首页首次文字需求能否提交。
- **可独立验证**：通过输入边界值即可验证，不依赖项目生成结果。
- **边界明确**：附件管理、项目创建与智能体执行均在其他任务中处理。
- **结论**：任务足够独立且原子，无需继续拆分。
