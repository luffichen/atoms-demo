# 浏览版本代码快照

## 目标

用户可以安全查看任一正式版本的文件树和文件内容，而不影响当前工作区或应用预览。

## 需求

1. “基于此版本查看代码”使文件查看器进入只读版本快照模式。
2. 服务端直接读取 Git tag 的 tree 和 blob，不 checkout 实际项目工作目录。
3. 查看器显示“正在查看 Vn”横幅和“返回当前代码”。
4. 快照模式支持文件树、文本、图片、大文件和二进制的现有只读呈现。
5. 多标签页可以同时查看不同历史版本，互不影响。
6. 应用预览、智能体工作目录和活动工作项不随历史浏览切换。
7. 历史快照只包含该版本受跟踪文件。

## 验收标准

- 浏览历史文件前后工作目录、分支和预览均不变化。
- 刷新页面可通过深链接恢复同一版本和文件。
- 无效 tag 或路径显示项目内错误状态。

## 不在本任务范围内

- 历史代码编辑。
- 历史版本预览。
- 文件下载。

## 独立性与原子性检查

- **单一职责**：只负责只读历史文件浏览。
- **可独立验证**：并行打开多个版本并监测工作区即可验证。
- **结论**：任务足够独立且原子。
